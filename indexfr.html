TIC TAC TOE

PROJET PYTHON 1 :

Le Tic Tac Toe est un jeu de stratégie classique qui a amusé des générations de joueurs. Il se joue sur une grille de 3x3, où les participants s'alternent pour remplir les cases avec leurs symboles respectifs — X ou O. L'objectif est clair : être le premier à aligner trois de ses symboles, que ce soit horizontalement, verticalement ou en diagonale.

Ce projet présente ma réalisation du Tic Tac Toe en Python, mettant en avant mes compétences en programmation et ma compréhension de la logique du jeu. Le jeu propose deux modes : un mode à deux joueurs où les utilisateurs peuvent s'affronter, et un mode bot où les joueurs peuvent défier un adversaire informatique. Dans ce portfolio, je vais explorer les différents composants du jeu, fournir des détails sur la structure du code, et partager des démonstrations vidéo du gameplay.

Fonctionnalités du jeu

Le Tic Tac Toe intègre les fonctionnalités essentielles suivantes pour enrichir l'expérience utilisateur :

Mode Deux Joueurs : Dans ce mode, deux joueurs peuvent s'affronter en s'alternant pour réaliser leurs mouvements. Le jeu invite les joueurs à indiquer la ligne et la colonne de leur choix, et le plateau se met à jour en temps réel. Après chaque mouvement, le jeu vérifie si l'un des joueurs a gagné, assurant ainsi une expérience captivante jusqu'à ce qu'un joueur remporte la partie ou que le jeu se termine par un match nul.

Mode Bot : Le joueur a la possibilité d'affronter un bot, ce qui introduit un niveau de défi supplémentaire. Le bot applique une stratégie basique pour réaliser ses mouvements, en évaluant les occasions de victoire et en bloquant les actions du joueur lorsque cela est nécessaire. Ce mode offre aux joueurs l'opportunité de perfectionner leurs compétences face à un adversaire virtuel.

Décomposition du Code

La réalisation du Tic Tac Toe en Python repose sur plusieurs fonctions essentielles, chacune étant responsable d'une partie spécifique des fonctionnalités du jeu. Pour mieux illustrer les éléments clés du code, j'ai inclus plusieurs extraits de code mettant en évidence des sections importantes et leurs fonctions. Chaque extrait est accompagné d'une explication afin de clarifier son rôle dans l'ensemble de l'implémentation. Voici un aperçu des principaux composants du code :


# CREATE THE BOARD
def show_board(board):
    print(" ") 
    print(" " * 13, " TIC TAC TOE ", " " * 13)
    print(" ")
    print("Pour jouer, entrez le numéro de la ligne")
    print("et celui de la colonne. Exemple: 00") 
    print(" ")  
    print(" " * 13, " ", "0 ", " 1 ", " 2 ", " " * 13)
    print(" " * 13, "-" * 13, " " * 13)
    for index, row in enumerate(board): 
        print(" " * 11, index, "|", row[0], "|", row[1], "|", row[2], "|", " " * 13) 
        print(" " * 13, "-" * 13, " " * 13)
    print(" ")

Création du Plateau

La fonction show_board est chargée d'afficher l'état actuel du plateau de jeu aux joueurs. Elle imprime la configuration de la grille, incluant les chiffres correspondant à chaque cellule, et met à jour l'affichage après chaque mouvement. Cette fonction enrichit l'expérience utilisateur en offrant une représentation visuelle claire du jeu. 


#  CHECK FOR VICTORY
def victory(board, player): 
    for row in range(3): 
        if board[row][0] == player and board[row][1] == player and board[row][2] == player: 
            return True
    for col in range(3):    
        if board[0][col] == player and board[1][col] == player and board[2][col] == player:
            return True 
    if board[0][0] == player and board[1][1] == player and board[2][2] == player: 
        return True  
    if board[0][2] == player and board[1][1] == player and board[2][0] == player: 
        return True  
    return False

Vérification de la Victoire

La fonction victory détermine si un joueur a remporté la partie. Elle examine toutes les combinaisons gagnantes possibles : trois lignes, trois colonnes et deux diagonales. Si l'une de ces combinaisons présente le symbole du même joueur, la fonction renvoie True, signalant ainsi une victoire.


# BOARD FULL
def board_full(board):
    for row in board: 
        for cell in row:
            if cell == " ": 
                return False 
    return True

Contrôle du Plateau

La fonction board_full vérifie si le plateau de jeu est entièrement occupé. Elle parcourt toutes les cellules de la grille pour détecter la présence d'espaces vides. Si aucune cellule n'est vide, la fonction renvoie True, signalant ainsi un match nul. 


# PLAYER MOVE
def player_move(player_name, board): 
    while True:
        move = input(f"{player_name}, entrez votre mouvement (ligne colonne) : ")
        if len(move) == 2: 
            try:
                line = int(move[0]) 
                column = int(move[1])
                if 0 <= line <= 2 and 0 <= column <= 2:
                    if board[line][column] == ' ':
                        return line, column
                    else:
                        print("Cette case est déjà occupée. Essayez à nouveau.")
                else:
                    print("Veuillez entrer des chiffres entre 0 et 2.")
            except ValueError: 
                print("Entrée invalide. Veuillez entrer deux chiffres.")
        else:
            print("Entrée invalide. Assurez-vous d'entrer exactement deux chiffres.")

Logique du Mouvement du Joueur

La fonction player_move s'occupe de la saisie des mouvements des joueurs. Elle invite le joueur à entrer son coup sous la forme de numéros de ligne et de colonne. La fonction inclut une validation pour garantir que l'entrée est correcte, en vérifiant les longueurs et les plages appropriées, puis renvoie les coordonnées du mouvement.


# BOT MOVE
def bot_move(board, bot, human):
    if board[1][1] == ' ':
        return 1, 1
    for row in range(3): 
        for column in range(3):
            if board[row][column] == ' ':
                board[row][column] = bot
                if victory(board, bot):
                    return row, column  
                board[row][column] = ' ' 
    for row in range(3):  
        for column in range(3):
            if board[row][column] == ' ':
                board[row][column] = human
                if victory(board, human):
                    board[row][column] = ' '  
                    return row, column  
                board[row][column] = ' '  
    for row in range(3): 
        for column in range(3):
            if board[row][column] == ' ':
                return row, column 
    return -1, -1

Logique du Bot

La fonction bot_move met en œuvre la stratégie du bot pour effectuer des mouvements. Le bot commence par vérifier si la cellule centrale est libre ; si c'est le cas, il l'occupe. Ensuite, il analyse le plateau à la recherche d'opportunités de victoire ou de possibilités de bloquer le joueur humain. Si aucune de ces conditions n'est rencontrée, le bot sélectionne la première cellule disponible, assurant ainsi la continuité du jeu. 


# GAME PLAY FUNCTION
def game_tic_tac_toe(): 
    print(" ") 
    print(" " * 13, " TIC TAC TOE ", " " * 13)
    print(" ")
    while True:
        mode = input("Voulez-vous jouer contre un autre joueur (1) ou contre le bot (2)? Entrez 1 ou 2: ")    
        if mode == '1':
            player1_name = input("Entrez le nom du joueur 1 : ")
            player2_name = input("Entrez le nom du joueur 2 : ")
            break  
        if mode == '2':
            player1_name = input("Entrez votre nom : ")
            player2_name = "Bot"
            break 
        print("Choix invalide, veuillez entrer 1 ou 2.")  

    board = [[' ' for _ in range(3)] for _ in range(3)]
    actual_player = 'X'
    player_names = { 'X': player1_name, 'O': player2_name }
    
    while True:
        show_board(board)        
        if player_names[actual_player] == "Bot":
            print("Le bot réfléchit...")
            line, column = bot_move(board, 'O', 'X')
            if board[line][column] == ' ':
                board[line][column] = 'O'
                if victory(board, 'O'):
                    show_board(board)
                    print("Le bot a gagné !")
                    break
        else:
            line, column = player_move(player_names[actual_player], board) 
            board[line][column] = actual_player
        if victory(board, actual_player):
            show_board(board)
            print(f"Félicitations ! {player_names[actual_player]} a gagné !")
            break
        elif board_full(board):
            show_board(board)
            print("Match nul !")
            break

        actual_player = 'O' if actual_player == 'X' else 'X'

Boucle de Jeu

La fonction game_tic_tac_toe coordonne le déroulement du jeu. Elle commence par initier la partie en demandant aux joueurs de sélectionner leur mode (contre un autre joueur ou le bot) et met en place le plateau. Cette fonction inclut la boucle principale du jeu, qui alterne les tours entre les joueurs, vérifie les conditions de victoire ou de plateau plein, et met à jour l'état du jeu en conséquence.

Gestion des Erreurs

La gestion des erreurs est un élément essentiel du jeu, assurant une expérience utilisateur sans accroc et prévenant les plantages provoqués par des entrées invalides. Le jeu intègre divers mécanismes pour traiter les erreurs potentielles :

  • Validation des Entrées : La fonction player_move inclut une validation rigoureuse des saisies. Elle s'assure que l'entrée du joueur contient exactement deux caractères, et que ces deux caractères sont des chiffres. Si l'entrée est invalide, la fonction fournit un retour d'information clair, invitant le joueur à réessayer en respectant le format correct.
  • Vérification des Plages : Une fois que l'entrée a été confirmée comme numérique, la fonction vérifie si les numéros de ligne et de colonne fournis se situent dans la plage valide (de 0 à 2). Si les numéros dépassent cette plage, le joueur en est informé et invité à fournir des coordonnées valides.
  • Gestion des Cellules Occupées : Au cours de la partie, si un joueur essaie d'occuper une cellule déjà occupée, le jeu examine l'état du plateau et informe le joueur que la case choisie n'est pas disponible. Cela évite les mouvements invalides et assure que le jeu demeure équitable et agréable.

Ces mécanismes de gestion des erreurs favorisent une expérience utilisateur agréable, permettant aux joueurs de se concentrer sur le jeu sans être frustrés par des problèmes imprévus. En offrant des instructions et des retours d'information clairs, le jeu renforce l'engagement et le plaisir des participants.

Améliorations Possibles

En se projetant vers l'avenir, plusieurs améliorations potentielles pourraient être envisagées pour enrichir le jeu :

  • Optimisation de l'Intelligence du Bot : Actuellement, le bot adopte une stratégie simple, se concentrant sur la victoire ou le blocage des actions du joueur. Je souhaiterais intégrer un algorithme plus avancé, comme l'algorithme Minimax, qui permettrait au bot d'examiner tous les mouvements potentiels et de sélectionner le meilleur choix. Cela rendrait le jeu plus stimulant pour les joueurs.
  • Ajout d'une Interface Graphique (GUI) : Bien que l'interface textuelle soit opérationnelle, développer une interface graphique avec un framework tel que Tkinter ou Pygame améliorerait nettement l'expérience utilisateur. Une représentation visuelle du plateau de jeu la rendrait plus attrayante et intuitive pour les joueurs.
  • Implémentation de Différents Modes de Jeu : Je pourrais ajouter des modes de jeu supplémentaires, comme la possibilité de jouer contre un autre bot avec divers niveaux de difficulté, ou même un mode multijoueur où les joueurs peuvent s'affronter en ligne.
  • Statistiques et Classements : Intégrer une fonctionnalité pour suivre les victoires, les défaites et les matchs nuls des joueurs permettrait de leur fournir des retours sur leurs performances. De plus, un classement stimulerait l'esprit compétitif et offrirait un sentiment d'accomplissement.
  • Règles de Jeu Personnalisables : Autoriser les joueurs à modifier la taille du plateau (par exemple, des grilles de 4x4 ou 5x5) ou à changer les conditions de victoire (comme exiger quatre alignés) pourrait apporter de la diversité et améliorer la rejouabilité.

En intégrant ces améliorations, mon objectif est de développer un Tic Tac Toe plus solide, captivant et plaisant.

Améliorations Possibles

Pour améliorer la fonctionnalité et l'expérience utilisateur du Site de Fans, plusieurs améliorations peuvent être envisagées.

  • Optimisation des Médias et Minimisation du CSS : Améliorer les performances du site est une priorité essentielle, et cela peut être accompli en optimisant la gestion des médias et en réduisant la taille des fichiers CSS. La compression des images, ainsi que l'adoption de formats modernes tels que WebP, permet de diminuer considérablement les temps de chargement sans sacrifier la qualité visuelle.
    De plus, l'implémentation du chargement différé pour les images et vidéos garantit que seul le contenu visible est chargé au départ, ce qui contribue à un chargement de page plus rapide.
    En réduisant le nombre de feuilles de style, on minimise également les requêtes HTTP, ce qui entraîne un rendu de page plus rapide et offre une expérience utilisateur plus fluide et réactive.
  • Développement Backend pour des Fonctionnalités Dynamiques : Pour améliorer les fonctionnalités du Site de Fans, l'intégration du développement backend est essentielle. Cela permettra d'introduire des fonctionnalités dynamiques telles que la création de comptes utilisateurs, un contenu personnalisé et des abonnements sécurisés à la newsletter. En particulier, rendre la barre de recherche opérationnelle facilitera aux utilisateurs la recherche de contenu spécifique.
    Par ailleurs, le formulaire d'abonnement à la newsletter sera relié à un système backend pour assurer une gestion sécurisée des données des utilisateurs.
    De plus, le développement d'une section de commentaires fonctionnelle encouragera les utilisateurs à interagir et à partager leurs réflexions sur divers sujets, renforçant ainsi l'engagement au sein de la communauté.
    Enfin, l'implémentation d'un formulaire de contact simplifiera la communication avec les administrateurs du site, garantissant une gestion efficace des demandes et des retours.
